Trabalho API de COVID19

Esse trabalho utiliza da implementação de uma API em Kotlin para manutenção e acompanhamento dos casos de COVID19

Como usar

Login

Essa sessão é referente aos caminhos e métodos para o login:

Motivação Método Caminho Formato Observação
Realizar o login Post "/login" {
"email": "seuemail@server.com",
"password": "SUASENHA"
}		 O password padrão é "123456"
Realizar o logout Get "/logout"
Visualizar os usuários do sistema Get "/all-users" É necessário estar logado em uma conda de administrador
Trocar a senha Post "/mudar-senha" {
"email": "seuemail@server.com",
"password": "NOVASENHA"
}
Criar um usuário post "/login/cadastro" {
"cpf": "012345678901",
"email": "seuemail@server.com",
"password": "SUASENHA"
}		 Só cria usuário se não houver usuário com mesmo e-mail préviamente cadastrado
Acessar suas informações get "/informacoes-usuario" {
"email": "seuemail@server.com",
"password": "SUASENHA"
}

Consultas de Cidade

Essa sessão é referente aos caminhos e métodos para consultas de COVID19 em cidades:

Observação:

As funcionalidades dessa sessão só estão disponíveis após ser realizado o login no sistema

Motivação Método Caminho Formato Observação
Visualizar as 10 cidades com mais casos Get "/cidade/top-cases"
Visualizar as 10 cidades com mais mortes Get "/cidade/top-deaths"
Recuperar informações de determinada cidade Post "/cidade" {
"name": "Nome da Cidade",
"state": "Sigla do estado a que pertence"
}
Atualiza o número de casos de determinada cidade Post "/cidade/update-cases" {
"name": "Nome da Cidade",
"state": "Sigla do estado a que pertence"
"cases": "Int do número de casos a serem acrescentados"
}
Atualiza o número de mortes de determinada cidade Post "/cidade/update-deaths" {
"name": "Nome da Cidade",
"state": "Sigla do estado a que pertence"
"deaths": "Int do número de casos a serem acrescentados"
}
