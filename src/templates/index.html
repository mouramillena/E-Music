Trabalho API de COVID-19

Esse trabalho utiliza da implementação de uma API em Kotlin para atualização e acompanhamento dos casos de COVID-19

.:: MODO DE USAR ::.



:. Níveis de Acesso .:

Sessão referente ao controle dos Níveis de Acesso diponíveis na Aplicação:

Código Descrição Observação de Atuação
1 - Administrador Usuário MASTER da Aplicação - Realiza todas as Funções
2 - User Usuário com poderes de Alteração e Consulta - Responsável pela atualização das informações da Base de Dados
3 - Consultor Usuário com poderes de Consulta - Realiza apenas consultas à Base de Dados




:. Login .:

Sessão referente aos caminhos e métodos para realização do Login:

Descrição Nível de Acesso Método Controle de Acesso Caminho Formato Padrão Observação
Realizar o Login (P1) Todos os Níveis (1/2/3) POST "/login" {
"email": "seuemail@server.com",
"password": "SUASENHA"
}		 O password padrão é "123456"
Realizar o Logout (P1) Todos os Níveis (1/2/3) GET "/logout" ----- -----
Visualizar os usuários cadastrados no sistema (P1) Somente Admin (1) GET "/all-users" ----- É necessário estar logado em uma conta de administrador
Trocar a senha (P1) Todos os Níveis (1/2/3) POST "/mudar-senha" {
"email": "seuemail@server.com",
"password": "NOVASENHA"
}		 Para trocar a senha do usuário diferente do logado na sessão é necessário ser administrador!
Criar um usuário (P1) Somente Admin (1) POST "/login/cadastro" {
"cpf": "012345678901",
"email": "seuemail@server.com",
"password": "SUASENHA"
}		 Só é possível criar se estiver logado como administrador e só cria usuário se não houver usuário com mesmo e-mail préviamente cadastrado
Acessar suas informações (P1) Todos os Níveis (1/2/3) GET "/informacoes-usuario" ----- -----
Alterar Privilégios (P2) Somente Admin (1) POST "/mudar-log" {
"email": "email@server.com",
"tipo_log": 1/2/3
} 		Promovendo ou Revogando Permissões aos usuários

:. Atualizações nas Cidade .:

Essa sessão é referente aos caminhos e métodos para Atualizações nos Casos e Mortes de COVID-19 nas cidades: (Incremento e Decremento)

Observação: As funcionalidades desta sessão só estarão disponíveis após ser realizado o login no sistema
Descrição Nível de Acesso Método Caminho Formato Observação
Acrescentar o número de casos de determinada cidade (P1) Somente os Níveis (1/2) POST "/cidade/update-cases" {
"name": "Nome da Cidade",
"state": "Sigla do estado a que pertence"
"cases": "Int do número de casos a serem acrescentados"
}		 - Incremento -
Acrescentar o número de mortes de determinada cidade (P1) Somente os Níveis (1/2) POST "/cidade/update-deaths" {
"name": "Nome da Cidade",
"state": "Sigla do estado a que pertence"
"deaths": "Int do número de casos a serem acrescentados"
}		 - Incremento -
Decrementar o número de casos de determinada cidade (P2) Somente os Níveis (1/2) POST "/cidade/del-cases" {
"name": "Nome da Cidade",
"state": "Sigla do estado a que pertence"
"deaths": "Int do número de casos a serem decrementados"
}		 - Decremento -
Decrementar o número de mortes de determinada cidade (P2) Somente os Níveis (1/2) POST "/cidade/del-deaths" {
"name": "Nome da Cidade",
"state": "Sigla do estado a que pertence"
"deaths": "Int do número de casos a serem decrementados"
}		 - Decremento -
